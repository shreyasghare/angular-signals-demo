Signal Concepts

Angular Signals provide a new way to manage reactive state in Angular applications. Click on any concept below to learn more and see it in action.

signal()

Basic reactive state primitive. Create writable signals that can be read and updated.

computed()

Derived reactive values that automatically update when their dependencies change. Read-only signals.

Official Docs →

effect()

Side effects that run when signals change. Automatically tracks dependencies and runs when needed.

Official Docs →

model()

Two-way binding signal for parent-child component communication. Combines input and output.

Official Docs →

input()

Component input signal. Declare component inputs as signals for reactive updates.

Official Docs →

output()

Component output event signal. Emit events from child to parent components.

Official Docs →

linkedSignal()

Writable signal derived from other signals but can be written to independently. Maintains reactivity while allowing overrides.

Official Docs →

resource() / rxResource()

Async reactivity with resources for handling asynchronous data operations. Incorporate async data into signal-based code.
