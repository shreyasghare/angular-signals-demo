linkedSignal()

Writable signal derived from other signals but can be written to independently

Overview

The linkedSignal() function creates a writable signal that is derived from other signals but can be written to independently. It maintains reactivity to source changes while allowing manual overrides.

Usage

import {{ '{' }} signal, linkedSignal {{ '}' }} from '@angular/core';

sourceValue = signal('Hello');

// Create a linked signal
linkedValue = linkedSignal<string, string>({{ '{' }}
  source: this.sourceValue,
  computation: (sourceValue, previous) => {{ '{' }}
    // If source changed, use source value
    // If linked signal was written to directly, keep previous value
    return previous && previous.value !== sourceValue 
      ? previous.value as string 
      : sourceValue;
  {{ '}' }}
{{ '}' }});

// Can be read and written
const value = linkedValue();
linkedValue.set('Custom Value');

Key Features

  • Writable - can be updated independently of the source
  • Reactive - automatically reacts to source signal changes
  • Computation function - determines how to merge source and manual changes
  • Useful for editable computed values or validated inputs

Example in Demo

In the Signals Demo, linked signals are used for editable full names (can override computed values), validated age (keeps last valid value), and linked display values (can be edited independently).
