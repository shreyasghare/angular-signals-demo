resource() / rxResource()

Async reactivity with resources for handling asynchronous data operations

Overview

The resource() function creates a resource that incorporates async data into your application's signal-based code. rxResource() is a wrapper around HttpClient that provides request status and response as signals, making HTTP requests through the Angular HTTP stack.

Usage

import {{ '{' }} resource {{ '}' }} from '@angular/core';
import {{ '{' }} rxResource {{ '}' }} from '@angular/core/rxjs-interop';
import {{ '{' }} HttpClient {{ '}' }} from '@angular/common/http';

// Using resource() with fetch
const userId = signal('123');
const userResource = resource({{ '{' }}
  params: () => ({{ '{' }} id: userId() {{ '}' }}),
  loader: ({{ '{' }} params {{ '}' }}) => fetchUser(params)
{{ '}' }});

// Using rxResource() with HttpClient
const postsResource = rxResource({{ '{' }}
  request: () => ({{ '{' }} {{ '}' }}),
  loader: () => this.http.get<Post[]>('https://api.example.com/posts')
{{ '}' }});

// Access resource value
const posts = computed(() => postsResource.value() ?? []);
const isLoading = computed(() => postsResource.isLoading());
const error = computed(() => postsResource.error());

Key Features

  • Async data handling - incorporate async operations into signal-based code
  • Reactive - automatically reloads when params change
  • Status signals - provides isLoading, error, and value signals
  • HTTP integration - rxResource works seamlessly with Angular HttpClient
  • Request cancellation - automatically aborts requests when params change
  • Type-safe - TypeScript types are enforced for request and response data
